# Plan: Add api_base support and test against remote Ollama

**Date**: 2026-03-23
**Goal**: Allow agents to specify a custom LLM API base URL in their YAML config, then run a live test against the Ollama instance at http://192.168.2.107.

---

## Changes

### Step 1 — config.py: add optional `api_base` field

```python
class AgentConfig(BaseModel):
    ...
    api_base: str | None = None   # e.g. "http://192.168.2.107:11434"
```

No validation needed — litellm accepts any string or None.

### Step 2 — cli.py: pass `api_base` to Provider

```python
provider = Provider(
    model=config.model,
    temperature=config.temperature,
    **({"api_base": config.api_base} if config.api_base else {}),
)
```

`Provider.__init__` already accepts `**kwargs` and forwards them to litellm — no changes needed there.

### Step 3 — agents/example.yaml: set api_base and model

```yaml
model: ollama/mistral
api_base: http://192.168.2.107:11434
```

Ollama's default port is 11434. If the remote instance runs on a different port we can adjust.

### Step 4 — Live test

```bash
python3 -m agent99.cli run agents/example.yaml --input "Say hello in one sentence."
```
