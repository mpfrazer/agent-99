# Plan: Implement loop.py

**Goal**: Implement the core agent loop — think → act → observe, repeat — that ties together AgentConfig, Provider, and ToolRegistry.

---

## Context

The loop is the heart of the runner. It:
1. Builds the initial message list from the agent's system prompt and user input.
2. Calls the provider (think).
3. If the response contains tool calls, executes them and appends results (act + observe).
4. Repeats until the model returns a plain text response or `max_iterations` is reached.

---

## Design

### AgentLoop class

```python
class AgentLoop:
    def __init__(
        self,
        config: AgentConfig,
        provider: Provider,
        registry: ToolRegistry,
    ) -> None: ...

    def run(self, user_input: str) -> str:
        """Run the agent loop and return the final text response."""
```

### Message flow

```
messages = [
    {"role": "system", "content": config.system_prompt},   # omitted if empty
    {"role": "user",   "content": user_input},
]

loop:
    response = provider.complete(messages, tools=tool_schemas)

    if response.tool_calls:
        # append assistant message with tool_calls
        # for each tool call:
        #   execute tool, capture result (str) or error message
        #   append tool result message
        continue

    else:
        return response.content
```

### Tool execution

- Call `registry.get(name)(**arguments)`.
- If the tool raises, catch the exception and return an error string as the observation:
  `f"Error: {type(e).__name__}: {e}"` — the model sees the error and can decide how to proceed.
- Tool results are always converted to `str` (call `str()` on the return value).

### Iteration limit

If `max_iterations` is reached without a final text response, raise `RuntimeError("max_iterations reached")`.

### Tool schemas

Only pass schemas for the tools listed in `config.tools`. If `config.tools` is empty, pass no tools (pure text completion).

### Message format for tool calls (OpenAI format, litellm-compatible)

```python
# Assistant message with tool calls
{
    "role": "assistant",
    "content": None,
    "tool_calls": [
        {
            "id": tc.id,
            "type": "function",
            "function": {"name": tc.name, "arguments": json.dumps(tc.arguments)},
        }
        for tc in response.tool_calls
    ],
}

# Tool result message
{
    "role": "tool",
    "tool_call_id": tc.id,
    "content": result_str,
}
```
