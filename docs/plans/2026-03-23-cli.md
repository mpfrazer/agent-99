## Commands

### `agent99 run <agent_file> [--input TEXT]`

Run an agent from a YAML file.

- Load `AgentConfig.from_yaml(agent_file)`
- Build `Provider(model=config.model, temperature=config.temperature)`
- Build `ToolRegistry`, auto-register `tools/filesystem.py` (always available)
- Build `AgentLoop(config, provider, registry)`
- If `--input` is given, pass it directly to `loop.run()`
- If `--input` is not given, read from stdin (supports piping: `echo "..." | agent99 run agent.yaml`)
- Print the final response to stdout
- On error (ValueError, RuntimeError), print the message to stderr and exit with code 1

### `agent99 new <name>`

Scaffold a new agent YAML file.

- Write `agents/<name>.yaml` with sensible defaults (model: ollama/mistral, no tools)
- Print the path of the created file
- Error if the file already exists

### `agent99 list`

List all `.yaml` files in the `agents/` directory relative to cwd.

- Print one file per line (just the filename, no path)
- If the directory doesn't exist or is empty, print a friendly message

### `agent99 tools`

List all tools registered by default (i.e. from `tools/filesystem.py`).

- Print each tool name and its one-line description
- Format: `  read_file    Read the contents of a file and return them as a string.`

---

## Design decisions

- Use `typer.Typer()` with `app = typer.Typer()` — the entry point in `pyproject.toml` already points to `agent99.cli:app`.
- No `invoke_without_command=True` — each command is explicit.
- Errors use `typer.echo(..., err=True)` + `raise typer.Exit(code=1)` rather than `sys.exit` for testability.
- The `run` command auto-discovers tools from `tools/filesystem.py` by importing the module and calling `registry.register_module`. Future tool modules can be added to a fixed list in cli.py.
- `new` writes to `agents/<name>.yaml` relative to cwd — simple, no config file needed.
