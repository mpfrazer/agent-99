# Plan: Implement registry.py

**Date**: 2026-03-23
**Goal**: Auto-discover plain Python tool functions and generate OpenAI-compatible JSON schemas from their type hints and docstrings — no decorators required.

---

## Context

Agents reference tools by name in their YAML config (`tools: [read_file, list_dir]`).
The registry is responsible for:
1. Loading tool functions from Python modules (e.g. `tools/filesystem.py`)
2. Generating the JSON schema litellm/OpenAI expect for tool calling
3. Resolving tool names → callable at runtime

---

## Design

### Schema generation rules

Given a plain function like:

```python
def read_file(path: str, encoding: str = "utf-8") -> str:
    """Read the contents of a file.

    Args:
        path: Absolute or relative path to the file.
        encoding: File encoding (default utf-8).
    """
```

The registry produces:

```json
{
  "type": "function",
  "function": {
    "name": "read_file",
    "description": "Read the contents of a file.",
    "parameters": {
      "type": "object",
      "properties": {
        "path":     {"type": "string", "description": "Absolute or relative path to the file."},
        "encoding": {"type": "string", "description": "File encoding (default utf-8)."}
      },
      "required": ["path"]
    }
  }
}
```

Rules:
- Parameters with a default value are NOT in `required`.
- Parameters without a default are in `required`.
- Type mapping: `str → "string"`, `int → "integer"`, `float → "number"`, `bool → "boolean"`, anything else → `"string"` (safe fallback).
- Description comes from the first line of the docstring (everything before the first blank line).
- Per-parameter descriptions are parsed from a Google-style `Args:` block if present; empty string otherwise.
- `return` annotation and `**kwargs`/`*args` are ignored.

### ToolRegistry class

```python
class ToolRegistry:
    def register(self, fn: Callable) -> None: ...
    def register_module(self, module: ModuleType) -> None:
        # registers all public callables in the module
    def get(self, name: str) -> Callable: ...          # raises KeyError if missing
    def schemas(self, names: list[str]) -> list[dict]: # returns schemas for named tools
    def all_schemas(self) -> list[dict]: ...
```

### Standalone helper

```python
def build_schema(fn: Callable) -> dict: ...
```

Exposed at module level so loop.py / tests can use it directly without a registry instance.

### Step 3 — Write tests/test_registry.py

Cover:
1. `build_schema` on a simple function — name, description, parameters, required fields.
2. Type mapping for str, int, float, bool, unknown type.
3. Parameters with defaults excluded from `required`.
4. Google-style `Args:` docstring → per-param descriptions.
5. Function with no docstring — description is empty string, params have no description.
6. `ToolRegistry.register` / `get` round-trip.
7. `ToolRegistry.register_module` picks up all public callables.
8. `ToolRegistry.schemas` returns schemas only for named tools.
9. `ToolRegistry.get` raises `KeyError` for unknown tool.
10. Schema for `read_file` from `tools/filesystem.py` is valid.

### Step 4 — Verification

`pytest tests/test_registry.py -v` — all pass.
